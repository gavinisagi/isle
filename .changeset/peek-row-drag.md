feat(host): draggable island from the peek state. The whole peek (hover-collapsed) pill row is a drag surface, so the island can be repositioned without expanding: a `mousedown` on the row starts a JS drag session (new `drag-start/-move/-end` IPC) that moves the window by tracking the OS cursor in DIP (`getCursorScreenPoint`); a sub-threshold (~4px) press falls through to click→expand, so dragging and click-to-expand coexist without `-webkit-app-region: drag` (which would swallow the click). Supersedes an earlier tiny-handle attempt (too small to aim at). Also fixes two collapse bugs: blur-driven collapse now clears `hovered` (an expanded island clicked-away no longer lands in peek); a fast drag no longer snaps back to the pill's original position (a `dragging` flag locks peek + keeps the window interactive, so the cursor outrunning the window can't drop hover, shrink the pill, and trigger a re-centering resize).
